基本流程之四： 频率和带宽检测

频率和带宽检测主要分一下几个部分：

  1. 搜索中心频点
    • 搜索中心频点的算法是相当复杂的算法，这个算法因各个芯片厂商而不同，协议中并没有详细的规定。
    • 根据UE物理层实现，UE可能测试RSSI并且决定是否进入下一步。
  2. 检测PSS（主同步序列）
  3. 检测SSS（次同步序列）
    • 如果UE正确译码PSS和SSS，则UE可以可以获得小区ID（所以如果你在UE的追踪日志里找到了小区ID，并且这个小区ID与其中的eNodeB匹配，说明UE成功的检测了PSS和SSS）
  4. 根据第2)步和第3)步的结果，UE可以检测CRS（小区参考信号）
  5. 根据中心频点的72个子载波（6RB）译码BCH。
  6. BCH（MIB）告知系统的频率信息（比如系统频率带宽）
MasterInformationBlock ::=SEQUENCE {
    dl-Bandwidth ENUMERATED { n6, n15, n25, n50, n75, n100},
    phich-Config PHICH-Config,
    systemFrameNumber BIT STRING (SIZE (8)),
    spare BIT STRING (SIZE (10))
